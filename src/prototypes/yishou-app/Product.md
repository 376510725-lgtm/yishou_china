## 1. 执行摘要

### 1.1 产品定位

**益寿巴渝**是一款为中老年人群（50岁+）量身定制的移动社交与生活服务平台，以重庆本地文化为情感纽带，提供便民服务、健康管理、社交互动、兴趣社群和积分激励等一体化服务。产品以「温暖、简单、实用」为设计哲学，解决中老年人在城市生活中的社交匮乏、信息不对称和服务获取困难等问题。

### 1.2 产品愿景

让每一位长辈都能在指尖上享受便捷的社区生活，在有温度的社交圈中找到归属感。

### 1.3 产品核心价值

| 价值主张 | 说明 |
|----------|------|
| 一站式便民服务 | 覆盖助餐、助浴、助医等七助服务 + 社区生活（赶场、聚会、赛事） |
| 有温度的社交 | 基于熟人/同城的视频流、兴趣圈子、实时聊天 |
| 陪伴式健康管理 | 养生助手、万年历、养老社区等适老化健康工具 |
| 积分激励体系 | 观看内容、参与互动获得积分，可兑换实物商品与优惠券 |
| 本地化运营 | 以重庆巴渝文化为核心场景的社区拼购、赶场、赛事活动 |

## 2. 市场背景与机会

### 2.1 市场背景

- 中国 60 岁以上人口已突破 2.97 亿（2025），老龄化进程加速
- 重庆作为老龄化程度较高的直辖市，60 岁以上人口占比超过 20%
- 中老年人智能手机普及率持续攀升，但市场上缺乏真正适老化的社交+生活一站式 APP
- 现有竞品侧重单一功能（社交或广场舞），综合服务能力弱

### 2.2 机会点

- **空白市场**: 尚未出现兼具便民服务、社交、健康管理的综合平台
- **区域深耕**: 以重庆作为试点，形成「巴渝」品牌壁垒后再向其他城市复制
- **家庭纽带**: 子女为父母安装/代操作的意愿强，可通过家庭裂变获客

### 2.3 竞品参照

| 竞品 | 优势 | 劣势 |
|------|------|------|
| 微信/支付宝 | 用户基数大 | 功能繁杂，适老化不足 |
| 糖豆/老友圈 | 广场舞内容丰富 | 缺乏生活服务和本地化运营 |
| 社区养老平台 | 服务专业 | 社交属性弱，用户粘性低 |

益寿巴渝的核心差异在于 **服务+社交+激励** 三位一体，且专注重庆本地运营。

## 8. 核心用户流程

- **便民服务**: 启动 > 登录 > 益寿Tab > 点击助餐 > 浏览 > 预约 > 填写 > 确认
- **社交互动**: 启动 > 登录 > 巴渝Tab(默认) > 熟人视频流 > 点赞/评论 > 积分弹窗 > 继续
- **积分兑换**: 我的Tab > 积分商城 > 浏览分类 > 点击商品 > 选择规格 > 兑换
- **拼单购买**: 益寿Tab > 土货拼购 > 浏览活动 > 选商品 > 立即购买 > 提订单 > 支付

## 9. 数据模型

### 9.1 核心实体关系

`
User (用户) --1:N-- Post (帖子)
User --1:N-- FollowRelation (关注关系)
User --1:N-- ChatMessage (聊天消息)
User --1:N-- PointsRecord (积分兑换记录)
User --1:N-- Order (拼单订单)
Post --N:1-- Circle (圈子)
Order --N:1-- PindanProduct
PointsRecord --N:1-- PointsGoods
`

### 9.2 关键接口

**UserProfile**: { id, name, avatar, color, uid, ip, bio, following, followers, likesReceived, isFollowed }

**FollowPost**: { id, userId, user, avatar, avatarColor, type('image'|'video'), coverSeed, coverHeight, title, likes, comments?, saves?, shares?, date?, images? }

**PindanOrder**: { id, orderNo, status('pending'|'paid'|'shipped'|'completed'|'cancelled'), totalAmount, address, contact, phone, products[], createdAt }

**PointsGoods**: { id, name, points, cash?, remaining, category('gift'|'health'|'hobby'|'coupon'), stock, desc, image, images[], specs?, purchaseNotes?, afterSalesNotes? }

### 9.3 数据来源

- 当前版本所有数据为 **Mock Data**，硬编码在 index.tsx 中
- 预置约 30 用户、50+ 帖子、16 个圈子、21 件积分商品、4 条赛事、2 个拼购活动、7 个拼购商品、3 条订单
- 后续版本需对接后端 API 实现数据动态加载
