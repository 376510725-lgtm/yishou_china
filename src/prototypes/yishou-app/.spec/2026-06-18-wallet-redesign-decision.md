# 我的钱包改版设计决策

日期：2026-06-18

## 用户选择

采用 B 方案：账户分层型。

## 产品需求

钱包重新定义为：

```text
钱包 = 资金账户（可提现和不可提现）
     + 积分账户（平台奖励积分）
     + 数字资产包（优惠券、会员权益）
     + 数据权益中心（数据贡献值、数据收益记录）
```

## 设计决策

1. 首页首屏优先突出资金账户，保留用户对“钱包”的传统理解。
2. 资金账户内明确拆分可提现余额和不可提现余额，避免用户误解。
3. 积分账户、数字资产包、数据权益中心作为二级资产入口呈现。
4. 数字资产包需要承接优惠券、会员权益、平台兑换资格和到期提醒。
5. 数据权益中心用独立模块表达数据贡献值、累计收益、收益记录和授权管理。
6. 视觉继续沿用益寿巴渝当前适老化风格：橙色主品牌、浅背景、大字重信息层级。

## 本轮不做

1. 不接入真实支付、提现或收益结算接口。
2. 不新增复杂数据授权流程，只做原型层面的入口和说明。
3. 不改变“我的”页整体结构，只重做“我的钱包”页面及必要二级页入口。
