## 1. 产品概述

### 1.1 产品定位

益寿巴渝是一款专为 **50 岁以上中老年人群** 打造的 UGC 社交与生活服务移动端应用。产品以重庆地域文化（巴渝）为情感锚点，集便民服务（六助体系）、健康管理、社交广场、兴趣社群、积分商城、土货拼购于一体。

**一句话描述**：全国首个以地域文化为锚点、以「社交 + 生活服务 + 健康管理」三位一体为核心的一站式老年智慧生活 App。

### 1.2 产品愿景与使命

**愿景**：成为中国最具温度的老年智慧生活服务平台，让每一位长者都能享受数字时代的便利与温暖，实现「老有所依、老有所乐、老有所为」。

**使命**：
- **连接**：连接长者与社区、长者与家庭、长者与优质生活服务，打破数字鸿沟和社交孤岛
- **赋能**：以适老化科技赋能中老年群体，降低数字使用门槛
- **守护**：通过健康管理、紧急救助、用药提醒等功能，为长者的健康安全提供全天候守护
- **激发**：通过社交广场、兴趣社群、内容创作，激发长者的社交活力与创造力

### 1.3 品牌调性

| 项目 | 内容 |
|------|------|
| APP 名称 | 益寿巴渝 |
| 品牌色 | 暖橙红 `#FF6B35` |
| 情感锚点 | 重庆地域文化（巴渝山水、渝中老街、码头文化） |
| 宣传口号 | 「记录美好生活 · 分享给家人朋友」 |
| 设计哲学 | 零学习成本、零心理负担、零操作障碍 |

### 1.4 核心价值主张

益寿巴渝解决的根本问题是：中国 5 亿+ 50 岁以上中老年人在数字化时代面临的「三重断裂」——

| 断裂类型 | 具体表现 | 益寿巴渝解决路径 |
|----------|----------|-----------------|
| **社交断裂** | 退休后社交圈缩小，子女不在身边，孤独感强烈 | 巴渝广场 + 霍圈子构建线上社交家园 |
| **服务断裂** | 需要生活服务但不知道去哪找、不知道可信不可信 | 六助服务体系 + 平台信任背书 + 评价机制 |
| **数字断裂** | 想用手机但功能太复杂，担心被骗 | 极致适老化设计 + AI 语音交互 + 安全守护 |

| 序号 | 竞争壁垒 | 说明 | 可持续性 |

## 2. 市场与用户分析

### 2.1 市场概况

> 详细市场分析见《益寿巴渝 APP — 市场调研报告》。以下为关键摘要。

- **目标市场**：银发经济，2026 年规模约 12 万亿人民币
- **目标用户基数**：全国 50 岁以上人口约 5 亿，重庆市 50 岁以上人口约 800 万
- **政策窗口期**：2024-2027 年为最佳切入期，政策密集出台

## 4. 功能需求详述

### 4.1 登录与注册

**屏幕标识**：`login`  
**优先级**：P0  
**设计目标**：降低注册门槛，保障账号安全，适配老年人操作习惯。

#### 功能描述

登录页支持 **4 种登录模式** 切换（`LoginMode`），用户可根据自身情况选择最便捷的方式。

| 模式 | 核心交互 | 关键规则 |
|------|----------|----------|
| `main` | 登录首页：Logo + APP 名称 + 宣传语（「记录美好生活 · 分享给家人朋友」） + 三大圆角入口按钮 | 首次进入或退出登录后展示 |
| `phone-sms` | 手机号输入（+86 前缀，11 位）+ 短信验证码（6 位，60 秒倒计时） + 协议勾选 → 登录 | 验证码发送间隔 ≥ 60s，每日上限 5 次 |
| `password` | 手机号 + 密码（显隐切换图标） + 协议勾选 → 登录；底部「忘记密码」→ 3 步弹窗（验证手机号 → 设置新密码 → 完成） | 密码需 8-16 位，含数字和字母 |
| `register` | 手机号 + 邀请码（选填） + 验证码（60s 倒计时） + 密码（8-16 位，含数字和字母）+ 协议勾选 → 注册 | 手机号唯一性校验（模拟） |

#### 验收标准

- [ ] 4 种登录模式可正常切换，不丢失已输入信息
- [ ] 所有登录方式未勾选协议时，登录按钮置灰不可点击
- [ ] 短信验证码 60 秒倒计时正常，倒计时结束可重新获取
- [ ] 忘记密码 3 步流程完整：验证手机号 → 设置新密码 → 成功提示
- [ ] 第三方授权绑定流程完整：授权后绑定手机号
- [ ] 协议全文可在弹窗中完整查看

#### 适老化适配

- 输入框高度 ≥ 56px，字体 ≥ 17sp
- 验证码输入支持语音播报（规划中）
- 协议文字提供「大字版摘要」（规划中）
- 按钮均有 active 按下态反馈（scale:95%）

### 4.3 便民服务（六助体系）

#### 4.3.1 助餐服务

**屏幕标识**：`service`  
**优先级**：P0  
**设计目标**：帮助老年人快速找到、筛选并预约附近的餐饮服务。

**页面构成**：

### 5.2 导航机制

  - 益寿下子页面 → 返回益寿工作台

## 11. 成功指标与验收标准

### 11.1 核心 KPI

| 维度 | 指标 | 年度目标 | 监测频率 |
|------|------|----------|----------|
| 用户增长 | 月活用户数（MAU） | 年增长 ≥ 150% | 月度 |
| 用户粘性 | 次月留存率 | ≥ 55% | 月度 |
| 用户粘性 | 日均使用时长 | ≥ 25 分钟 | 周度 |
| 用户活跃 | DAU/MAU 比值 | ≥ 35% | 周度 |
| 内容生态 | 日均 UGC 发帖量 | 年增长 ≥ 100% | 周度 |
| 服务交易 | 月均助老服务订单量 | 年增长 ≥ 200% | 月度 |
| 商业转化 | GMV（拼购+商城+服务） | 年增长 ≥ 180% | 月度 |
| 用户满意度 | NPS（净推荐值） | ≥ 50 | 季度 |
| 服务品质 | 服务订单好评率 | ≥ 90% | 周度 |
| 供给质量 | 活跃服务商数量 | 年增长 ≥ 150% | 月度 |

### 11.2 MVP 验收总标准

- [ ] 全部 P0 功能完成开发并通过测试
- [ ] 核心任务完成率 ≥ 90%（预约助餐/浏览视频/用药提醒/拼购下单）
- [ ] 首屏加载时间 ≤ 3 秒（4G 网络）
- [ ] 无严重/阻塞性 Bug
- [ ] 适老化合规检查通过（工信部标准）

### 11.3 季度 OKR 框架示例

**O1：打造最易用的适老化生活服务入口**
- KR1：六助服务覆盖率达重庆主城区老年人口的 30%
- KR2：服务订单完成率 ≥ 95%，用户好评率 ≥ 90%
- KR3：平均服务响应时间 ≤ 30 分钟

**O2：构建活跃的老年社交生态**
- KR1：巴渝广场 DAU 突破 5 万
- KR2：霍圈子聊天室日均活跃圈子 ≥ 50 个
- KR3：人均日内容消费时长 ≥ 15 分钟

**O3：建立可持续的商业模型**
- KR1：月度 GMV 突破 200 万元
- KR2：签约优质服务商数量 ≥ 500 家
- KR3：会员付费转化率 ≥ 3%

## 12. 风险与假设

### 12.1 关键假设

| 假设 | 验证方式 | 优先级 |
|------|----------|--------|
| 50 岁以上用户愿意使用 UGC 社交产品 | MVP 数据验证（DAU/内容量/留存率） | 高 |
| 子女会帮助父母注册和使用 APP | 注册来源 / 邀请链路数据分析 | 高 |
| 六助服务是老年人的真实刚需 | 服务下单量 / 复购率 | 高 |
| 巴渝地域文化能有效建立品牌认同 | 用户调研 + NPS 评分 | 中 |
| 服务商愿意入驻平台 | 服务商入驻申请量 / 活跃度 | 中 |

### 12.2 主要风险与应对

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|----------|
| 目标用户数字素养不足，推广困难 | 中 | 高 | 社区地推 + 老年大学合作 + 子女邀请裂变 |
| 服务供给质量参差不齐 | 高 | 高 | 服务商评级体系 + 定期巡检 + 用户评价机制 |
| 互联网巨头进入银发赛道 | 中 | 高 | 加速建立社交关系链 + 文化壁垒 + 服务供应链 |
| 老年人数据安全舆情风险 | 中 | 高 | 严格合规体系 + 透明隐私政策 + 最小必要收集 |
| 用户增长不及预期 | 中 | 中 | 多渠道获客 + 内容预置 + 策略灵活调整 |
| 政策细则变化 | 低 | 中 | 保持政策沟通 + 提前建立合规体系 |
