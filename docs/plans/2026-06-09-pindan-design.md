# 土货拼购功能设计方案

## 日期
2026-06-09

## 一、产品定位

土货拼购是益寿平台的独立电商模块，主打来自农村/本地的土特产（如土鸡蛋、土蜂蜜、腊肉、农家蔬菜等），以"拼购活动"为主要展示形式，先到先得抢完即止。

## 二、目标用户

- **主要用户**：老年人，喜欢土特产、追求健康原生态
- **辅助用户**：子女帮父母下单

## 三、整体结构

```
益寿Tab → 土货拼购入口 → 土货拼购主页
                              ↓
                        活动专场列表
                              ↓
                    点击专场 → 专场详情页
                              ↓
                    商品列表 → 点击商品
                              ↓
                        商品详情页
                              ↓
                       确认订单 → 支付（支付宝）
```

## 四、页面清单

| 页面 | 说明 |
|------|------|
| 土货拼购主页 | 展示活动专场列表（专场卡片形式） |
| 专场详情页 | 展示该专场下的所有商品 |
| 商品详情页 | 商品图文介绍、拼购价、立即拼购按钮 |
| 确认订单页 | 选择收货地址、确认商品和数量 |
| 订单列表页 | 我的订单，支持筛选状态 |
| 订单详情页 | 查看订单状态、物流等信息 |

## 五、订单状态

- 待付款
- 待收货
- 已完成
- 已取消

## 六、支付方式

- 支付宝

## 七、页面详细设计

### 7.1 土货拼购主页

- 顶部标题栏："土货拼购"
- 活动专场卡片列表：
  - 专场封面图
  - 专场名称（如"春季农家好物"）
  - 专场描述
  - 进度信息（剩余名额 / 总名额）
  - 点击进入专场详情

### 7.2 专场详情页

- 顶部返回按钮 + 专场名称
- 专场banner图
- 商品列表（卡片形式）：
  - 商品图片
  - 商品名称
  - 原价（划线）
  - 拼购价
  - 已抢进度条
  - 点击进入商品详情

### 7.3 商品详情页

- 商品轮播图
- 商品名称
- 原价（划线） + 拼购价
- 商品描述/详情图文
- 数量选择器
- "立即拼购"按钮

### 7.4 确认订单页

- 商品信息（图片、名称、数量、单价）
- 收货地址选择
- 配送方式
- 订单金额汇总（商品金额 + 运费）
- "提交订单"按钮 → 跳转支付宝支付

### 7.5 订单列表页

- Tab筛选：全部 / 待付款 / 待收货 / 已完成 / 已取消
- 订单卡片：
  - 订单编号、时间
  - 商品信息
  - 订单金额
  - 订单状态
  - 操作按钮（如"去支付"、"确认收货"）

### 7.6 订单详情页

- 订单状态展示
- 商品列表
- 收货地址
- 支付信息
- 物流信息（如已发货）

## 八、数据结构

### 活动专场 (PindanActivity)
- id, title, description, coverImage, totalQuota, remainingQuota, startTime, endTime, status

### 商品 (PindanProduct)
- id, activityId, name, image, originalPrice, pindanPrice, description, stock

### 订单 (PindanOrder)
- id, orderNo, userId, status, totalAmount, address, products, createdAt, paidAt

## 九、UI设计原则

- 大按钮、大字体，适合老年人操作
- 清晰的进度提示（如剩余名额）
- 简化操作流程，减少决策点
- 保持与现有益寿App风格一致
