# 开发服务器架构说明

本文档介绍项目开发模式下的服务器架构和启动指南。

---

## 一、架构概览

项目使用两个独立的服务协同工作：

```
┌─────────────────────────────────────────────────────────────┐
│                     你的 Windows 电脑                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────────────────┐      ┌────────────────────────┐ │
│   │   Vite Dev Server    │      │    Make Server         │ │
│   │   (客户端开发服务器)   │      │   (管理服务/后台API)     │ │
│   │                      │      │                        │ │
│   │   端口: 51720        │      │   端口: 53817          │ │
│   │   IP: 0.0.0.0        │      │   IP: 0.0.0.0          │ │
│   │                      │      │                        │ │
│   │   负责:              │      │   负责:                │ │
│   │   - 原型页面预览      │      │   - 页面热重载同步       │ │
│   │   - HMR 热模块替换    │      │   - 实时编辑功能         │ │
│   │   - 编译 TS/React    │      │   - 组件属性编辑         │ │
│   │   - 处理图片/资源     │      │   - 预览状态通知         │ │
│   │                      │      │                        │ │
│   │   直接面向用户        │      │   被页面内嵌引用         │ │
│   └──────────────────────┘      └────────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、两个服务的关系

| 服务 | 端口 | 作用 | 是否必须 |
|------|------|------|----------|
| **Vite** | 51720 | 编译并运行原型页面 | ✅ 必须 |
| **Make Server** | 53817 | 提供开发辅助工具（热同步、实时编辑等） | ❌ 可选 |

### 工作流程

当你访问 `http://172.16.21.146:51720/prototypes/yishou-app` 时：

1. **Vite (51720)** 返回 HTML 页面
2. HTML 中内嵌引用 Make Server 的脚本：
   ```html
   <script src="http://172.16.21.146:53817/runtime/quick-edit.js"></script>
   <script src="http://172.16.21.146:53817/assets/dev-template-bootstrap.js"></script>
   ```
3. 浏览器从 **53817** 加载预览编辑功能的代码

### 类比理解

- **51720 (Vite)** = 餐厅前台服务员 → 直接给你上菜（页面内容）
- **53817 (Make Server)** = 厨房监控系统 → 在后台帮你看到热更新状态、实时编辑功能

## 三、端口说明

### 51720 - Vite 开发服务器

- **作用**：主 Web 服务器，负责编译和提供原型页面
- **启动命令**：`npm run dev` 或 `npm start`
- **监听地址**：0.0.0.0（可被局域网访问）
- **依赖**：Node.js 环境

### 53817 - Make Server

- **作用**：提供开发辅助功能（预览同步、实时编辑等）
- **启动方式**：通过 `npx @axhub/make` 或启动脚本
- **监听地址**：0.0.0.0（可被局域网访问）
- **注意**：这是内部服务，不直接面向用户

## 四、启动指南

### 方式一：启动 Vite 开发服务器（必须）

```bash
cd yishou_china
npm run dev
```

输出示例：
```
  VITE v5.4.21  ready in 3860 ms

  ➜  Local:   http://localhost:51720/
  ➜  Network: http://172.16.21.146:51720/
```

### 方式二：同时启动 Make Server（可选）

Make Server 提供预览编辑功能，需要额外启动：

1. 启动 Vite：`npm run dev`
2. 启动 Make Server：`npx @axhub/make`
3. Make Server 监听在 53817 端口

或使用项目中提供的启动脚本：
- `start-both.bat` - 同时启动 Vite 和 Make Server
- `start-admin-server.bat` - 仅启动 Make Server

### 停止服务

在终端按 `Ctrl+C` 可以停止服务。

## 五、局域网访问说明

### 可用地址

服务器运行后会显示多个可访问地址：

```
  ➜  Local:   http://localhost:51720/         # 仅本机可访问
  ➜  Network: http://172.16.21.146:51720/    # 局域网可访问 ⚠️
  ➜  Network: http://172.22.192.1:51720/     # 其他网络接口
```

### 网络地址说明

| IP 地址 | 网络类型 | 访问范围 |
|----------|----------|----------|
| 127.0.0.1 | Loopback | 仅本机 |
| 172.27.16.1 | Default Switch (Hyper-V) | 仅本机 |
| 172.16.21.146 | WLAN 适配器 | 局域网可访问 ✅ |
| 172.22.192.1 | 其他虚拟网络 | 特定网络内 |

### 访问规则

- **本机访问**：使用 `localhost:51720` 或 `127.0.0.1:51720`
- **局域网其他设备**：使用 WLAN IP，如 `http://172.16.21.146:51720`
- **注意**：Default Switch (172.27.x.x) 是 Hyper-V 虚拟网络，外部设备无法访问

## 六、常见问题

### Q: 为什么 localhost 能访问，但局域网 IP 访问是空白页？

检查以下几点：
1. 确认使用的是 WLAN IP（如 172.16.21.146）而不是 Default Switch IP（如 172.27.16.1）
2. 检查远程设备是否和服务器在同一 WiFi 网络
3. 检查 Windows 防火墙是否允许端口 51720 和 53817 入站

### Q: Make Server 不启动会影响页面访问吗？

不会。Make Server 是可选的辅助功能，用于提供预览编辑能力。不启动它，页面基本功能仍然正常，只是少了编辑辅助功能。

### Q: 如何确认两个服务都在运行？

在命令行执行：
```powershell
netstat -ano | Select-String "51720\|53817" | Select-String "LISTEN"
```

应该看到两个端口都在监听。
