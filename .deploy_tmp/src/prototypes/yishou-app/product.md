## 1. 产品概述

### 1.1 产品定位

专为 **50 岁以上中老年人群**打造的 UGC 社交与生活服务移动端 APP。集便民服务（六助+社区生活）、健康管理、社交广场、兴趣社群、积分商城、土货拼购于一体。品牌以重庆本地文化（巴渝）为情感锚点，以「界面极简、操作友好、温暖亲切」为核心设计原则。

### 1.2 目标用户

| 维度 | 描述 |
|------|------|
| **年龄** | 50 岁以上，主力 55-75 岁 |
| **数字能力** | 从基础（能接打电话）到熟练（能发朋友圈），覆盖不同层次 |
| **核心诉求** | 社交陪伴（广场舞/太极/钓鱼）、健康管理（养老社区/养生/用药提醒）、生活便利（助餐/助医/助行）、兴趣学习（书法/手工/摄影） |
| **操作偏好** | 大字、大按钮、少步骤、高对比度 |

### 1.3 品牌调性

| 项目 | 内容 |
|------|------|
| **APP 名称** | 益寿巴渝 |
| **品牌色** | 暖橙红 `#FF6B35` |
| **情感锚点** | 重庆地域文化（巴渝山水、渝中老街、码头文化） |
| **宣传口号** | 「记录美好生活 · 分享给家人朋友」 |
| **模拟设备** | 手机 390×844px |

## 3. 信息架构与导航

### 3.1 底部 Tab 栏（主导航）

| Tab | 标识 | 图标 | 说明 |
|-----|------|------|------|
| 益寿 | `yishou` | AppstoreOutlined | 工作台，便民服务+健康入口 |
| 巴渝 | `bayu` | FireOutlined | 社交广场，**默认首页** |
| 消息 | `messages` | MessageOutlined（带 15 角标） | 聊天+通讯录+通知 |
| 我的 | `profile` | UserOutlined | 个人中心+设置+积分 |

> **注意**：首次打开应用默认进入巴渝（`activeTab` 初始值 `'bayu'`），底部 Tab 视觉顺序为益寿→巴渝→消息→我的。

### 3.2 导航栈机制

- 所有子页面使用**栈式导航**（push/pop），从 main 进入子页面前保存当前 tab 状态
- 返回使用 `goBack()` → 从栈中 pop 出上一屏，恢复对应的 tab 和子 tab
- 进入聊天室时保存当前巴渝子 tab 状态，返回时恢复至对应子 tab

### 3.3 全局返回规则

| 当前页面 | 返回目标 |
|----------|----------|
| 益寿下的任意子页面 | 益寿工作台 |
| 巴渝下的任意子页面 | 巴渝广场（恢复对应子 tab） |
| 消息下的任意子页面 | 消息列表 |
| 我的下的任意子页面 | 我的页面 |
| 其他任意子页面 | 上一页（或主界面） |

## 4. 功能模块详述

### 4.1 登录与注册

登录页支持 **4 种登录方式切换**（`LoginMode`）：

| 模式 | 页面名称 | 核心流程 |
|------|----------|----------|
| `main` | 登录首页 | Logo + APP 名称 + 宣传语 + 三大入口按钮 |
| `phone-sms` | 短信验证码登录 | +86 前缀 + 11 位手机号 + 验证码（60s 倒计时） + 协议勾选 |
| `password` | 密码登录 | 手机号 + 密码（显隐切换） + 忘记密码入口 + 协议勾选 |
| `register` | 注册 | 手机号 + 邀请码（**必填**） + 验证码 + 密码（8-16 位） + 协议勾选 |

**关键交互**：
- 所有登录方式需勾选「用户协议」和「隐私政策」后方可提交
- 忘记密码弹窗：3 步弹窗流程（验证手机号 → 设置新密码 → 完成）
- 第三方授权绑定：模拟第三方登录后绑定手机号
- 协议全文：支持弹窗查看

### 4.3 便民服务（六助）

六助服务是益寿巴渝的**核心生活服务模块**，覆盖老年人日常生活的主要场景。

#### 4.3.1 助餐（`service`）

- **页面构成**：Hero Banner（橙色渐变）+ 筛选 Tab + 餐厅卡片列表 + 底部拨打电话按钮
