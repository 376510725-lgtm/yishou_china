## 1. 执行摘要

### 1.1 问题陈述

中国 50+ 中老年群体面临三大痛点：
- **社交孤独**：子女离家后社交圈萎缩，缺乏同好交流渠道
- **生活不便**：助餐/助浴/助洁/助医等日常服务获取渠道分散
- **数字鸿沟**：现有 APP 操作复杂、字体小、流程多，学习成本高

### 1.2 解决方案

打造一款以重庆本地文化为情感锚点、专为中老年人设计的 UGC 社交与生活服务移动端 APP——**益寿巴渝**。集便民服务（六助 + 社区生活）、健康管理、兴趣社交、积分商城、土货拼购于一体。

### 1.3 成功指标

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 日活跃用户（DAU） | MVP 阶段 ≥ 5,000 | 埋点统计 |
| 用户留存率（次日） | ≥ 40% | 行为分析 |
| 功能完成率 | 核心流程 ≤ 3 步到达目标页 | 路径分析 |
| 用户满意度（NPS） | ≥ 50 分 | 问卷调研 |
| 服务商入驻数 | MVP 阶段 ≥ 50 家 | 后台统计 |

### 1.4 产品定位

> 专为 **50 岁以上中老年人群**打造的「社交 + 生活服务」一站式平台。品牌口号：**「记录美好生活 · 分享给家人朋友」**

### 3.2 登录与注册

#### 3.2.1 功能描述

支持 4 种登录/注册模式切换：

| 模式 | 标识 | 必填字段 | 流程说明 |
|------|------|----------|----------|
| 登录首页 | `main` | 无 | Logo + 宣传语 + 三大入口按钮 |
| 短信验证码 | `phone-sms` | 手机号 + 验证码 | +86 前缀 + 11 位手机号 + 60s 倒计时 |
| 密码登录 | `password` | 手机号 + 密码 | 密码显隐切换 + 忘记密码入口 |
| 注册 | `register` | 手机号 + **邀请码（必填）** + 验证码 + 密码 | 邀请制注册，不可跳过 + 密码 8-16 位 |

#### 3.2.2 关键交互规则

> **邀请制说明**：本 APP 采用**邀请制注册机制**，新用户必须输入有效的邀请码才能完成注册。邀请码由已注册用户在「我的」页面生成并分享给亲友。

- 所有登录方式必须勾选「用户协议」和「隐私政策」后方可提交
- 注册时**邀请码为必填项**，需校验有效性（格式/是否已使用/是否存在）
- 忘记密码采用 **3 步弹窗流程**：验证手机号 → 设置新密码 → 完成
- 第三方授权绑定：模拟第三方登录后绑定手机号（两步流程）
- 协议全文通过弹窗展示，不可跳过

#### 3.2.3 验收标准

- [ ] 4 种登录模式可无缝切换，状态不丢失
- [ ] 注册流程中**邀请码为必填字段**，空值提交时报错提示
- [ ] 邀请码校验：8-16 位字符，支持字母数字组合
- [ ] 邀请码有效性验证：调用接口检查是否存在、是否可用
- [ ] 验证码倒计时 60s，期间不可重复发送
- [ ] 密码强度校验：8-16 位，包含字母和数字
- [ ] 第三方授权流程完整可走通

#### 3.3.1 助餐（`service`）

### 5.2 可用性要求（适老化设计）

| 规范项 | 数值 | 设计理由 |
|--------|------|----------|
| 基础字体大小 | **16px** | 保证中老年人群可读性 |
| 最小字体大小 | **12px** | 仅用于辅助说明文字 |
| 按钮最小高度 | **≥ 56px** | 便于手指点击 |
| 触摸目标尺寸 | **≥ 60px × 60px** | 适合老年人手指操作 |
| 卡片圆角 | **16px (rounded-2xl)** | 柔和视觉感受 |
| 按钮圆角 | **28px (rounded-2xl)** | 大圆角更友好 |
| 弹窗/Sheet 圆角 | **24px (rounded-3xl)** | 顶层浮层柔和感 |
| 行距 | **1.8 倍** | 提升长文本可读性 |
| 操作反馈 | **active:scale-95** | 按下时缩小反馈 |
| 颜色对比度 | **WCAG AA 标准** | 文字与背景对比度 ≥ 4.5:1 |
| 模拟设备尺寸 | **390 × 844px** | iPhone 14 尺寸原型展示 |

## 10. 风险与约束

### 10.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 单文件过大（index.tsx ~1MB） | 维护难度高、编译慢 | 高 | 后续考虑拆分为模块化组件 |
| 模拟数据与真实 API 差异 | 联调时接口对接问题 | 中 | 提前定义 API 契约文档 |
| 图片依赖 picsum.photos | 国内访问不稳定 | 中 | 替换为 CDN 或本地资源 |

### 10.2 业务风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 服务商入驻意愿低 | 六助服务供给不足 | 中 | 提供入驻补贴和流量扶持 |
| 用户内容质量参差不齐 | 社区氛围受损 | 中 | 加强内容审核和举报机制 |
| 中老年用户推广难 | 获客成本高 | 高 | 线下地推 + 子女推荐激励 |

### 10.3 合规约束

- 《个人信息保护法》合规：手机号脱敏、权限最小化原则
- 《互联网信息服务算法推荐管理规定》：排名算法透明可解释
- 无障碍设计参考：GB/T 37668-2019《信息技术 互联网内容无障碍可访问性技术要求与测试方法》
- 适老化设计参考：《互联网应用适老化及无障碍改造专项行动方案》
