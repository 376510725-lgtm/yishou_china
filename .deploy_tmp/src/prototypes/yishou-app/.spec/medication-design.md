## 1. 问题背景与用户需求

### 1.1 用户痛点

50 岁以上中老年人普遍存在慢性病用药需求（高血压、糖尿病、关节炎等），调研发现：

| 痛点 | 普遍性 | 风险等级 |
|------|--------|----------|
| 忘记按时服药 | 约 60% 慢病老人在一个月内至少漏服 1 次 | 高 |
| 混淆多种药物的服用时间 | 同时服用 3 种以上药物的老人中约 40% | 高 |
| 不记得是否已经服过药（重复/漏服） | 约 30% | 高 |
| 子女无法远程了解父母服药情况 | 子女不在身边的家庭中约 70% | 中 |
| 不同药物有不同饭前/饭后要求 | 仅 50% 能准确说出每种药的关系 | 中 |

### 1.2 目标用户场景

| 场景 | 用户 | 发生频率 | 核心需求 |
|------|------|----------|----------|
| 早晨起床后服用降压药 | 张大爷（68岁，高血压） | 每天 | 准时提醒，记录已服 |
| 饭后服用降糖药 | 刘奶奶（72岁，糖尿病） | 每天3次 | 区分饭前饭后 |
| 每周一注射一次胰岛素 | 孙阿姨（67岁，糖尿病） | 每周 | 固定周期提醒 |
| 感觉膝盖疼时服用止痛药 | 陈大叔（69岁，关节炎） | 按需 | 非固定时间记录 |
| 子女帮父母设置用药计划 | 子女（远程协助） | 一次性+调整 | 操作简单，可远程 |

## 2. 设计方案总览

### 2.1 功能范围

| 功能 | MVP | v2 | 说明 |
|------|-----|-----|------|
| 今日用药时间线 | v | | 按时间排序展示今日所有药品及状态 |
| 一键标记已服 | v | | 点击药品卡片标记为已服，有确认反馈 |
| 已服/待服/漏服状态 | v | | 不同颜色和图标区分三种状态 |
| 今日概览统计 | v | | 顶部卡片显示已服/待服/漏服数量 |
| 添加药品 | v | | 填写药品名、用量、服用时间、饭前/饭后 |
| 编辑/删除药品 | v | | 长按或点击进入编辑 |
| 按时推送提醒 | v | | 模拟/本地通知提醒 |
| 漏服提醒 | | v | 超过时间 30 分钟未标记则标为漏服 |
| 服药历史日历 | | v | 按月查看服药记录和依从率 |
| 药品图鉴/百科 | | v | 常见药品信息参考 |
| 子女远程关注 | | v | 绑定家庭成员查看记录 |

### 2.2 整体页面结构

```
用药提醒（全屏 Modal / 独立页面）
├── 导航栏（返回按钮 + "用药提醒" 标题 + 添加按钮）
├── 今日概览卡片（已服 / 待服 / 漏服 统计）
├── 时间线药物列表
│   ├── 时间段标题（早晨/上午/下午/晚上/睡前）
│   ├── 药品卡片（按时间排序）
│   │   ├── 状态指示区（圆点 + 勾/叹号）
│   │   ├── 药品信息（名称 + 用量 + 服用说明）
│   │   ├── 时间行（时间 + 饭前/饭后标签）
│   │   └── 操作按钮（已服 / 跳过）
│   └── 空状态（"还没有添加药品，点击下方按钮添加"）
└── 添加药品弹窗
    ├── 药品名称（输入）
    ├── 用量（输入：如 "1片" / "10ml"）
    ├── 服用时间（时间选择器 + 多时段支持）
    ├── 服用方式（Tab 选择：空腹/饭前/饭后/睡前/随餐）
    ├── 服药频率（每日 / 每周X次 / 按需）
    └── 备注（选填）
```

### 2.3 主题色

| 元素 | 色值 | 用途 |
|------|------|------|
| 主题色 | `#1677FF` | 品牌蓝色，Banner/按钮/高亮 |
| 浅背景 | `#E6F4FF` | 卡片/区域背景 |
| 已服色 | `#52C41A` | 绿色，已完成 |
| 待服色 | `#1677FF` | 蓝色，等待中 |
| 漏服色 | `#FF4D4F` | 红色，已错过 |
| 背景色 | `#F5F4F0` | 页面背景 |

## 3. 页面布局与交互详述

### 3.1 导航栏

```
[ ← ]          用药提醒          [ + ]
```

- 返回按钮: `ArrowLeftOutlined`，点击返回益寿工作台
- 标题: 17px 700w 居中
- 添加按钮: `PlusOutlined` 或 `FormOutlined`，点击弹出添加药品弹窗

### 3.2 今日概览卡片

```
┌─────────────────────────────────┐
│  💊 今日用药                      │
│                                  │
│  ✅ 已服 3     ⏳ 待服 2    ⚠️ 漏服 1 │
│                                  │
│  今日依从率: 60%  [  ██████░░░░  ]  │
└─────────────────────────────────┘
```

- 圆角: 16px (rounded-2xl)
- 背景: 蓝色渐变 `linear-gradient(135deg, #1677FF 0%, #4096FF 100%)`，白色文字
- 三个统计数字并排显示，各带图标+颜色区分
- 依从率进度条（灰色底 + 蓝色进度）

### 3.3 时间线药物列表

按时间段分组，每个药品为一个圆角卡片：

```
    早晨                          ← 时间段标题
  ┌─────────────────────────────────┐
  │ ◉ 8:00                           │
  │ 氨氯地平片        1片/次          │
  │ 降压药 · 已服                     │
  │ [ 已服 ✓ ]                       │ → 绿色按钮，不可再点
  └─────────────────────────────────┘

  ┌─────────────────────────────────┐
  │ ○ 8:30                           │
  │ 二甲双胍片        0.5g/次         │
  │ 降糖药 · 饭后                    │
  │ [ 标记已服 ]                     │ → 蓝色按钮，点击变绿色
  └─────────────────────────────────┘

    上午
  ┌─────────────────────────────────┐  ← 漏服状态（红色）
  │ ✕ 12:30                          │
  │ 阿卡波糖片        50mg/次         │
  │ 降糖药 · 随餐                     │
  │ ⚠️ 已漏服                        │ → 红色突出显示
  └─────────────────────────────────┘

    下午
  ┌─────────────────────────────────┐
  │ ○ 17:30                          │
  │ 阿卡波糖片        50mg/次         │
  │ 降糖药 · 随餐                     │
  │ [ 标记已服 ]                     │
  └─────────────────────────────────┘
```

**交互细节**:

| 操作 | 反馈 |
|------|------|
| 点击「标记已服」按钮 | 按钮变绿色 + 显示 ✓ + 药品左侧圆点变绿色实心勾 + 概览统计更新 |
| 点击已服药品卡片 | 进入药品编辑弹窗（可修改用量/时间） |
| 左滑药品卡片 | 显示「删除」按钮（红色） |
| 长按药品卡片 | 震动反馈 + 弹出操作菜单（编辑/删除） |
| 点击概览卡的统计数字 | 快速筛选显示对应状态（如点击"漏服"只显示漏服药品） |
| 药品超过设定时间 30 分钟未标记 | 自动转为漏服状态（红色 "+30min" 标签） |

### 3.4 添加/编辑药品弹窗

```
┌─────────────────────────────────────────┐
│         添加药品                          │
│  ─────────────────────────────────────── │
│                                          │
│  药品名称: [________________]            │
│                                          │
│  每次用量: [_____] [片/粒/ml/支 ▼]       │
│                                          │
│  服用时间:                               │
│  ┌──────────────────────┐               │
│  │  08:00               [ ✕ ]          │
│  │  12:30               [ ✕ ]          │
│  │  [ + 添加时间 ]                      │
│  └──────────────────────┘               │
│                                          │
│  服用方式:                               │
│  [空腹] [饭前] [饭后] [随餐] [睡前]      │
│                                          │
│  备注: [___________________________]     │
│  (如：避免与柚子同服)                     │
│                                          │
│  ┌──────────────────────┐               │
│  │    保存药品           │               │
│  └──────────────────────┘               │
└─────────────────────────────────────────┘
```

**时间选择器**: 采用 iOS 风格滚轮时间选择，点击「添加时间」弹出底部选择器

**方式选择**: 单选 Tab 按钮组，选中状态蓝色填充

**剂量单位**: 常见单位快捷选项：片、粒、ml、支、包、丸

### 3.5 空状态

当用户尚未添加任何药品时：

```
   💊 （大图标，ClockCircleOutlined 48px，半透明蓝色）
   
   还没有定时用药
   
   点击右上角 "+" 添加你的第一种药品
   设置后每天准时收到服药提醒
```

## 4. 数据模型

```typescript
// 药品条目
interface Medication {
  id: number;
  name: string;             // 药品名称
  dosage: string;           // 用量（如 "1片"）
  dosageUnit: string;       // 用量单位（片/粒/ml/支/包/丸）
  times: string[];          // 服用时间数组（如 ["08:00", "20:00"]）
  timing: 'pre-meal' | 'post-meal' | 'empty-stomach' | 'bedtime' | 'with-meal';
  // 服用方式：饭前/饭后/空腹/睡前/随餐
  note?: string;            // 备注
  frequency: 'daily' | 'weekly' | 'as-needed';
  // 频率：每日/每周/按需
  daysOfWeek?: number[];    // 每周时指定星期几（1=周一 ... 7=周日）
  color?: string;           // 区分色（可选）
  createdAt: string;
}

// 每日服药记录
interface DailyRecord {
  date: string;             // 日期 "2026-06-10"
  medications: {
    medicationId: number;
    medicationName: string;
    time: string;           // 计划服用时间
    status: 'pending' | 'taken' | 'missed';
    // 待服/已服/漏服
    takenAt?: string;       // 实际服用时间
  }[];
}

// 预置示例数据（MVP 展示用）

    times: ['21:00'], timing: 'bedtime',

];
```

## 5. 与现有设计规范的一致性

### 5.1 视觉一致性

| 现有规范 | 用药提醒方案 |
|----------|-------------|
| 品牌色 `#FF6B35` | 保留，只用于概览卡片的 accent 点缀 |
| 主题色 `#1677FF` (蓝色) | 蓝色系，与 HEALTH_TOOLS 定义一致 |
| 基础字体 16px | 沿用 |
| 按钮高度 >= 56px | 主操作按钮 56px |
| 卡片圆角 16px | 所有卡片 16px (rounded-2xl) |
| 弹窗圆角 24px | 添加/编辑弹窗 24px (rounded-3xl) |
| 触摸目标 >= 60px | 所有可点击区域 >= 60px |
| active:scale-95 | 所有按钮添加该反馈 |
| PAGE_BG `#F5F4F0` | 内容区域背景沿用 |
| 导航栏 54px 白色底 | 沿用 |
| 标题 17px 700w 居中 | 沿用 |

### 5.2 交互一致性

| 现有模式 | 用药提醒方案 |
|----------|-------------|
| 返回使用 goBack() | 沿用 |

## 6. 开发范围（MVP vs v2）

### 6.1 MVP 实现范围

| 组件 | 实现方式 |
|------|----------|
| 页面容器与导航 | 标准子屏容器（54px 导航栏 + 可滚动内容区） |
| 今日概览卡片 | 蓝色渐变卡片 + 三个统计数字 + 进度条 |
| 时间线药物列表 | 按时间段分组 + 药品卡片 |
| 三种状态展示 | 绿色(已服)/蓝色(待服)/红色(漏服) |
| 标记已服交互 | 点击按钮切换为已服状态 |
| 添加药品弹窗 | 药品名/用量/时间/方式/备注 |
| 漏服自动判定 | 超过设定时间 30 分钟后自动变红 |
| 空状态 | 无药品时的引导页面 |
| 模拟数据 | 3 条预置药品（降压药+降糖药+阿司匹林） |

### 6.2 v2 扩展方向

| 功能 | 说明 |
|------|------|
| 服药日历 | 按月查看历史记录 |
| 依从率统计 | 周/月依从率图表 |
| 推送通知 | 浏览器 Notification API or 本地模拟 |
| 多用户/家庭绑定 | 子女可添加并关注 |
| 药品交互提醒 | 两种药物冲突时提示 |
| 扫码添加药品 | 扫描药盒条形码 |
| 语音提醒 | 播报药品名称和用量 |

## 7. 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|----------|
| 功能使用率 | 访问益寿Tab用户中 > 30% 点击用药提醒 | 埋点统计 |
| 药品设置完成率 | 打开页面用户中 > 60% 添加至少 1 种药 | 弹窗提交率 |
| 标记依从率 | 已设药品种 > 70% 按时标记 | 标记/计划比 |
| 单次操作时长 | 标记已服 < 3 秒 | 点击路径耗时 |
| 用户满意度 | > 4.0/5.0 | 内测问卷 |
